El comando **`rm` (remove)** es, sin duda, la herramienta más peligrosa y respetada en el arsenal de un SysAdmin. No hay "Papelera de reciclaje" en la terminal de Linux: una vez que ejecutas `rm`, los datos desaparecen del sistema de archivos. Por eso, un profesional nunca lo usa a la ligera.

Aquí tienes la guía de **`rm`** para tu **`laboratorio02`**, estructurada por niveles de precaución:

---

### Nivel 1: El Junior (Borrado Simple y Seguro)

**1. Borrar un solo archivo:**
```bash
rm laboratorio02/etc/apache2/apache2.conf.bak
```
*   **Qué hace:** Elimina el archivo de respaldo especificado.
*   **Uso real:** Limpieza manual de archivos temporales o copias de seguridad viejas que ya no son necesarias después de una edición exitosa.

**2. Modo interactivo (`-i` de interactive):**
```bash
rm -i laboratorio02/var/www/html/index.php
```
*   **Qué hace:** El sistema se detiene y te pregunta: *¿Desea borrar el archivo? (y/n)*.
*   **Uso real:** **Protección extrema**. Es una excelente práctica cuando vas a borrar archivos críticos en producción para evitar un "error de dedo" catastrófico.

---

### Nivel 2: El Administrador Intermedio (Limpieza Selectiva)

**3. Borrar múltiples archivos con patrones (Globbing):**
```bash
rm laboratorio02/var/log/apache2/*.log
```
*   **Qué hace:** Borra todos los archivos que terminen en `.log` dentro de esa carpeta.
*   **Uso real:** Vaciar logs acumulados para liberar espacio en disco rápidamente.
*   **Peligro:** Si escribes un espacio accidental después del asterisco, podrías borrar mucho más de lo esperado.

**4. Borrar directorios vacíos (`-d`):**
```bash
rm -d laboratorio02/var/backups/configs/cuarentena
```
*   **Qué hace:** Solo borra la carpeta si está **completamente vacía**. Si tiene archivos, el comando fallará.
*   **Uso real:** Limpiar estructuras de directorios residuales sin riesgo de perder datos importantes que puedan haber quedado dentro.

---

### Nivel 3: El SysAdmin Senior (Borrado Recursivo y Forzado)

**5. Borrado recursivo (`-r` de recursive):**
```bash
rm -r laboratorio02/var/backups/configs/
```
*   **Qué hace:** Borra la carpeta y **todo su contenido** (archivos y subcarpetas).
*   **Uso real:** Eliminar un proyecto completo o una estructura de servicios que ha sido dada de baja.

**6. El "Modo Dios" (`-rf` - Recursive & Force):**
```bash
rm -rf laboratorio02/tmp/cache_vieja
```
*   **Qué hace:** Borra todo de forma recursiva y **no pregunta nada**, incluso si el archivo está protegido contra escritura o no existe.
*   **Uso real:** Limpiar cachés persistentes o directorios de aplicaciones bloqueadas. 
*   **Advertencia de SysAdmin:** Este comando es el causante de los peores desastres en servidores. **Nunca** lo ejecutes sin verificar tres veces la ruta que escribiste.

---

### Nivel 4: El "Gurú" del Sistema (Auditoría y Precisión)

**7. Borrado con confirmación visual (`-v` de verbose):**
```bash
rm -rv laboratorio02/etc/bind/zones/
```
*   **Qué hace:** Borra la carpeta y te muestra en tiempo real cada archivo que está siendo destruido: `removed 'archivo.conf'`.
*   **Uso real:** Auditoría de limpieza. En scripts de mantenimiento, el flag `-v` permite guardar en un log qué fue exactamente lo que se eliminó.

**8. El truco de seguridad "Slash-Check":**
Los SysAdmins senior a menudo escriben `ls` antes de `rm` para estar seguros:
```bash
# PASO 1: Ver qué se borraría
ls laboratorio02/var/log/exim4/*.1
# PASO 2: Si la lista es correcta, cambiar ls por rm
rm -v laboratorio02/var/log/exim4/*.1
```
*   **Uso real:** Evitar borrar archivos de producción por un error en el patrón de búsqueda (Wildcards). Es el protocolo de seguridad estándar en entornos críticos.
