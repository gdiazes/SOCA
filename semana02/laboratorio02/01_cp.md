### 2. Guía de `cp -v` (Nivel SysAdmin)

Ahora ejecuta estos comandos uno a uno. Verás que el sistema te responde confirmando cada acción.

#### Nivel 1: El Junior (Copia con confirmación visual)
```bash
cp -v laboratorio02/etc/apache2/apache2.conf laboratorio02/etc/apache2/apache2.conf.bak
```
*   **Salida esperada:** `'laboratorio02/etc/apache2/apache2.conf' -> 'laboratorio02/etc/apache2/apache2.conf.bak'`
*   **Análisis:** Tienes la certeza absoluta de que el backup se creó.

#### Nivel 2: El Intermedio (Preservar y ver atributos)
```bash
cp -vp laboratorio02/etc/mysql/my.cnf.production laboratorio02/var/backups/configs/
```
*   **Qué hace `-vp`:** Copia, mantiene permisos/fechas y te avisa qué archivo movió.
*   **Uso real:** Respaldar configuraciones críticas de bases de datos manteniendo la integridad.

#### Nivel 3: El Senior (Recursividad Verbose)
Vamos a copiar toda la carpeta de configuración de Bind (DNS) a backups.
```bash
cp -vr laboratorio02/etc/bind/ laboratorio02/var/backups/configs/dns_backup
```
*   **Qué hace `-vr`:** Entra en la carpeta y te muestra una lista de cada archivo que va copiando. Es muy útil para carpetas con muchos archivos pequeños.

#### Nivel 4: El "Gurú" (Sincronización y Llaves con Verbose)

**A. El truco de las llaves (Brace Expansion):**
```bash
cp -vp laboratorio02/etc/exim4/exim4.conf{,.original}
```
*   **Resultado:** Verás `'laboratorio02/etc/exim4/exim4.conf' -> 'laboratorio02/etc/exim4/exim4.conf.original'`. ¡Ahorraste tiempo y el sistema te lo confirmó!

**B. Copia de actualización (`-u`):**
Intenta ejecutar este comando dos veces seguidas:
```bash
cp -vu laboratorio02/etc/cron.d/db_backup laboratorio02/var/backups/configs/
```
1.  **La primera vez:** Te dirá que lo copió.
2.  **La segunda vez:** No dirá nada (o dirá muy poco), porque el archivo en el destino ya es igual al del origen. El `-v` te confirma que el comando fue inteligente y no gastó recursos copiando de nuevo.
