El comando **`dd` (Data Duplicator)** es legendario en UNIX. Los administradores veteranos lo llaman cariñosamente **"Disk Destroyer"** (Destructor de Discos), porque si te equivocas en una sola letra, puedes borrar el disco duro principal del servidor de forma irrecuperable. 

A diferencia de `cp`, que copia *archivos*, `dd` copia **bloques de datos crudos (bytes)**. No le importan los sistemas de archivos; clona ceros y unos.

Aquí tienes la guía de **`dd`** para tu **`laboratorio02`**, desde crear archivos falsos hasta clonar sectores de arranque.

---

### Preparación del Escenario
Vamos a crear una carpeta aislada para jugar con `dd` sin romper nada crítico de tu laboratorio:
```bash
mkdir -p laboratorio02/var/backups/dd_data
echo "Super secreto" > laboratorio02/var/backups/dd_data/disco_virtual.img
```

---

### Nivel 1: El Junior (Copias y Archivos de Prueba)

La sintaxis de `dd` es única. Usa **`if`** (Input File / Origen) y **`of`** (Output File / Destino).

**1. Clonar un archivo (estilo bloque):**
```bash
dd if=laboratorio02/etc/apache2/apache2.conf of=laboratorio02/var/backups/dd_data/apache2.clone
```
*   **Qué hace:** Lee el archivo origen y escribe el destino byte por byte. Al final te muestra un resumen de cuántos bytes se copiaron.
*   **Uso real:** Aunque para esto es mejor `cp`, es la forma más básica de entender cómo fluye la data en `dd`.

**2. Crear un archivo de tamaño exacto (`bs` y `count`):**
```bash
dd if=/dev/zero of=laboratorio02/var/backups/dd_data/archivo_10MB.img bs=1M count=10
```
*   **Qué hace:** Lee del dispositivo virtual `/dev/zero` (que escupe ceros infinitamente) y escribe en un archivo. **`bs=1M`** significa "tamaño de bloque de 1 Megabyte" y **`count=10`** significa "hazlo 10 veces". Resultado: un archivo de exactamente 10MB.
*   **Uso real:** **Creación de archivos Swap** (memoria virtual) o para probar cuotas de disco de usuarios.

---

### Nivel 2: El Administrador Intermedio (Monitoreo y Destrucción)

**3. Ver el progreso (`status=progress`):**
Históricamente, `dd` no mostraba nada hasta terminar, dejando al SysAdmin sudando en frío preguntándose si el servidor se había colgado.
```bash
dd if=/dev/zero of=laboratorio02/var/backups/dd_data/archivo_50MB.img bs=1M count=50 status=progress
```
*   **Qué hace:** Muestra una barra de progreso en vivo, indicando la velocidad de escritura (MB/s).
*   **Uso real:** Clonar un disco de 1 Terabyte. Sin `status=progress`, estarías horas mirando una pantalla negra.

**4. Borrado Seguro de Datos (Wipe):**
```bash
dd if=/dev/urandom of=laboratorio02/var/backups/dd_data/disco_virtual.img bs=1K count=10
```
*   **Qué hace:** Lee de `/dev/urandom` (un generador de datos aleatorios) y sobrescribe el archivo destino con "basura" ilegible.
*   **Uso real:** **Destrucción forense de datos**. Si vas a desechar un disco duro de un servidor de base de datos, no basta con formatearlo. Un SysAdmin usa `dd` con urandom para sobrescribir todo el disco físico (`/dev/sdb`) con ruido, impidiendo la recuperación de datos por hardware.

---

### Nivel 3: El SysAdmin Senior (Cirugía de Discos y MBR)

Imagina que `archivo_50MB.img` es un disco duro real (`/dev/sda`). 

**5. Respaldar el MBR (Master Boot Record):**
El MBR es el sector que le dice al servidor cómo arrancar; ocupa exactamente los **primeros 512 bytes** del disco.
```bash
dd if=laboratorio02/var/backups/dd_data/archivo_50MB.img of=laboratorio02/var/backups/dd_data/mbr_backup.bin bs=512 count=1
```
*   **Qué hace:** Extrae única y exclusivamente los primeros 512 bytes del "disco" y los guarda a salvo.
*   **Uso real:** **Salvavidas antes de instalar un nuevo sistema operativo**. Si el nuevo SO rompe el arranque del servidor, puedes restaurar este pequeño archivo y el servidor volverá a bootear.

**6. Restaurar el MBR sin dañar los datos (`conv=notrunc`):**
```bash
dd if=laboratorio02/var/backups/dd_data/mbr_backup.bin of=laboratorio02/var/backups/dd_data/archivo_50MB.img bs=512 count=1 conv=notrunc
```
*   **Qué hace:** Vuelca los 512 bytes de vuelta al disco. **`conv=notrunc`** (No Truncar) es vital: le dice a `dd` que *solo* sobrescriba esos 512 bytes y deje intactos los otros 49.9 MB del archivo.
*   **Uso real:** Restaurar un sector de arranque sin borrar accidentalmente la tabla de particiones ni los datos de los clientes. Si omites `notrunc`, `dd` destruirá el resto del disco.

---

### Nivel 4: El "Gurú" del Sistema (Benchmarking I/O)

**7. Prueba real de velocidad de escritura (`oflag=dsync`):**
Los sistemas operativos mienten: cuando guardas un archivo grande, lo guardan en la memoria RAM (caché) y dicen "¡Terminé!" aunque el disco físico siga girando.
```bash
dd if=/dev/zero of=laboratorio02/var/backups/dd_data/test_io.img bs=1M count=100 oflag=dsync
```
*   **Qué hace:** `oflag=dsync` (Data Sync) obliga a `dd` a saltarse la memoria caché de Linux y escribir directamente en los platos magnéticos o chips del disco físico antes de contar el bloque como "copiado".
*   **Uso real:** **Medir el rendimiento real de un servidor (Benchmarking)**. Si un cliente se queja de que la Base de Datos va lenta, usas este comando para ver la velocidad real (MB/s) del disco duro sin que la RAM maquille los resultados.
