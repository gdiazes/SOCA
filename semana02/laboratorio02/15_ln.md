El comando **`ln` (link)** es la herramienta de "teletransportación" de archivos en Linux. Permite que un archivo o directorio aparezca en múltiples lugares del sistema al mismo tiempo sin duplicar el espacio en el disco duro.

Para un SysAdmin, existen dos tipos fundamentales de enlaces:
1.  **Enlaces Simbólicos (`-s` / Soft Links):** Son como "accesos directos". Apuntan a un nombre de ruta. Si borras el original, el enlace se rompe.
2.  **Enlaces Duros (Hard Links):** Apuntan directamente al **inodo** (el ADN del archivo). Si borras el nombre original, el archivo sigue vivo mientras exista al menos un enlace duro.

Aquí tienes la guía de **`ln`** para tu **`laboratorio02`**:

---

### Preparación del Escenario
Vamos a crear archivos específicos para probar la resistencia de los enlaces.

```bash
mkdir -p laboratorio02/var/www/v1 laboratorio02/var/www/v2
echo "Versión 1.0" > laboratorio02/var/www/v1/app.php
echo "Versión 2.0" > laboratorio02/var/www/v2/app.php

# Archivo para probar enlaces duros
echo "Datos críticos" > laboratorio02/etc/mysql/database.key

echo "Escenario para LN listo."
```

---

### Nivel 1: El Junior (El Acceso Directo / Soft Link)

**1. Crear un enlace simbólico a un archivo:**
```bash
ln -s laboratorio02/etc/apache2/apache2.conf laboratorio02/home/admin_shortcut.conf
```
*   **Qué hace:** Crea un "puntero" en la carpeta personal del administrador que redirige al archivo real de Apache.
*   **Uso real:** **Comodidad y rapidez**. No tienes que escribir rutas larguísimas como `/etc/apache2/sites-available/...` cada vez; creas un enlace en tu `home` para editarlo al instante.

---

### Nivel 2: El Administrador Intermedio (Enlaces Duros e Inodes)

**2. Crear un enlace duro (Hard Link):**
```bash
ln laboratorio02/etc/mysql/database.key laboratorio02/var/backups/configs/key_mirror
```
*   **Qué hace:** Crea un segundo nombre para el mismo contenido físico. Si ejecutas `ls -li`, verás que ambos archivos tienen el **mismo número de inodo**.
*   **Uso real:** **Protección contra borrado accidental**. Si un usuario borra el archivo original en `/etc/mysql/`, el archivo **NO se borra** del disco porque el enlace en `backups` sigue manteniendo los datos vivos. Es como tener un archivo con dos vidas.

---

### Nivel 3: El SysAdmin Senior (Gestión de Versiones)

**3. Sobrescribir un enlace existente (`-f` de force):**
Imagina que tu web apunta a la Versión 1, pero quieres pasar a la Versión 2.
```bash
# Creamos el enlace inicial a V1
ln -s var/www/v1/app.php laboratorio02/var/www/current_app

# Cambiamos a V2 "en caliente" (forzando el cambio)
ln -sf var/www/v2/app.php laboratorio02/var/www/current_app
```
*   **Qué hace:** La bandera `-f` elimina el enlace viejo y crea el nuevo en un solo paso atómico.
*   **Uso real:** **Despliegue de aplicaciones (Blue-Green Deployment)**. Tienes una carpeta llamada `current` que apunta a la versión estable. Cuando la nueva versión está lista, cambias el enlace simbólico. Si algo falla, vuelves a apuntar a la versión anterior en un milisegundo.

---

### Nivel 4: El "Gurú" del Sistema (Rutas Relativas y Directorios)

**4. Enlaces simbólicos a directorios y el peligro de las rutas:**
```bash
# Mal ejemplo (Ruta absoluta): ln -s /home/user/laboratorio02/var/log ...
# Buen ejemplo (Ruta relativa):
cd laboratorio02/var/www/html/
ln -s ../../log/apache2 logs_web
cd -
```
*   **Qué hace:** Crea un enlace dentro de la web que apunta a la carpeta de logs usando `../` para subir niveles.
*   **Uso real:** **Portabilidad**. Si usas rutas relativas, puedes mover toda la carpeta `laboratorio02` a otro servidor o a otro disco duro y los enlaces **seguirán funcionando**. Si usas rutas absolutas (que empiezan con `/`), el enlace se romperá al mover la carpeta.

**5. El flag `-n` (No seguir enlaces):**
```bash
ln -sfn laboratorio02/var/www/v1/ laboratorio02/var/www/current_dir
```
*   **Qué hace:** Evita que, al intentar actualizar un enlace que apunta a una carpeta, el sistema cree un enlace *dentro* de esa carpeta por error. Trata al enlace como si fuera un archivo normal.
*   **Uso real:** **Automatización de scripts**. Es obligatorio usar `-n` en scripts de Bash que actualizan carpetas de aplicaciones para evitar comportamientos extraños y bucles infinitos de carpetas.

---

### 🔥 Tu Reto de SysAdmin con `ln`

Tienes una base de datos pesada en un disco pequeño (`laboratorio02/var/lib/mysql/`) y te estás quedando sin espacio. Has instalado un disco nuevo gigante montado en `laboratorio02/mnt/disco_nuevo/`.

**Misión:**
1.  Mueve la carpeta de datos `empresa_db` al disco nuevo: 
    `mv laboratorio02/var/lib/mysql/empresa_db laboratorio02/mnt/disco_nuevo/` (simula esto creando las carpetas primero).
2.  Crea un **enlace simbólico** en la ubicación original (`laboratorio02/var/lib/mysql/empresa_db`) que apunte a la nueva ubicación en el disco gigante.
3.  Usa el flag **verbose** (`-v`) para confirmar la creación.
4.  Verifica con `ls -l` que el servicio MySQL ahora "cree" que sus datos siguen en el mismo lugar, aunque físicamente estén en otro disco.
